## ClearBoard - Drawing Fix

**Problem:** Drawing on the overlay wasn't working. Mouse events weren't being captured properly.

**Root cause:** NSHostingView doesn't forward mouse events to SwiftUI views. The `DrawingCanvasViewWrapper` was using `NSHostingView` which doesn't pass mouse events through to the wrapped SwiftUI view.

**Solution:**
1. Created `DrawingCanvasViewWrapper` as a pure `NSView` subclass
2. Implemented `mouseDown`, `mouseDragged`, `mouseUp` handlers directly
3. Draw elements using `CGContext` directly in `draw(_:)`
4. Fixed `acceptsFirstResponder` with `#if os(macOS)` guard
5. Made `smoothPoints()` public in DrawingManager
6. Added `@EnvironmentObject` to `DrawingElementView` for DrawingManager access

**Build errors fixed:**
- "expected '(' in argument list" - `acceptsFirstResponder` conflict between struct and NSView
- "smoothPoints is inaccessible due to private protection" - made method public

**Files modified:**
- `Sources/Drawing/DrawingCanvasView.swift` - rewritten with proper NSView wrapper
- `Sources/Drawing/DrawingManager.swift` - made `smoothPoints()` public

**Status:** BUILD SUCCEEDED - ready to test
